# PKS Revised AST (20260311) – Column Redundancy Fix

## Overview

The zip file `20260311.zip` contains a revised AST from PKS that eliminates column redundancy issues in DDS/file definitions. The pipeline is configured to use this AST by default.

## Setup

1. **Extract** (automatic on `./start_global_context_ui.sh`, or manual):
   ```bash
   python setup_ast_20260311.py
   ```
   This extracts to `JSON_ast/JSON_20260311/`.

2. **Verify efficiency**:
   ```bash
   python verify_ast_efficiency.py
   ```
   Compares old (JSON_20260227) vs new (JSON_20260311) AST:
   - Old: 59 duplicate columns across 23 table/column pairs
   - New: 0 duplicate columns

## Pipeline Usage

1. Start: `./start_global_context_ui.sh`
2. Click **Discover Directories** – JSON_ast/JSON_20260311 is selected by default
3. Build Global Context → Migrate Feature → Build Application

## AST Options

| AST Directory        | Description                          |
|----------------------|--------------------------------------|
| JSON_ast/JSON_20260311 | PKS revised (default, no column redundancy) |
| JSON_ast/JSON_20260227 | Previous AST (59 duplicate columns)  |
